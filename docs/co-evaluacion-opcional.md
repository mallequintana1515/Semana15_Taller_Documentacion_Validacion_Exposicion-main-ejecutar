# Coevaluacion opcional

| Aspecto revisado | Observacion | Recomendacion |
|---|---|---|
| Instalacion | El proyecto cuenta con `package.json` y se puede instalar con `npm install`. | Indicar siempre este comando en la documentacion de entrega. |
| Ejecucion | El proyecto puede ejecutarse con `npm run start` o abrirse con Live Server usando JSON Server aparte. | Recordar ejecutar `npm run db` para activar la base local. |
| Documentacion | La documentacion tecnica explica objetivo, tecnologias, estructura, pruebas y conclusiones. | Mantener las instrucciones claras para que otra persona pueda ejecutar el proyecto. |
| Bitacora | La bitacora registra pruebas, resultados, errores y correcciones aplicadas. | Agregar capturas en la carpeta de evidencias para respaldar cada prueba. |
| Evidencias | La estructura de carpetas permite guardar capturas de consola, pruebas, responsive y antes/despues. | Usar nombres claros para las imagenes, por ejemplo `01-proyecto-abierto.png`. |
| Exposicion | El guion organiza la presentacion en objetivo, tecnologias, demostracion, pruebas y aprendizaje. | Practicar una exposicion breve de 3 a 5 minutos. |
| Usabilidad | La pagina tiene un diseno mas profesional, botones claros y secciones organizadas. | Probar la pagina en celular para verificar que todos los elementos se lean bien. |
| Validaciones | El formulario valida campos obligatorios, correo, telefono, interes y comentario. | En una mejora futura, exigir exactamente 10 digitos para el telefono. |

## Comentario general

El proyecto esta bien organizado y cumple con el objetivo del taller porque no solo muestra una pagina funcionando, sino que tambien incluye validaciones, conexion con JSON Server, tabla de registros, grafico, documentacion y evidencias. La interfaz fue mejorada para verse mas profesional y la estructura de archivos facilita revisar cada parte del trabajo. Como mejora futura se recomienda agregar edicion y eliminacion de registros, filtros de busqueda y una validacion mas estricta del telefono.
