# Consultas de investigacion sugeridas

Antes de completar tu documentacion, busca informacion sobre estos temas y escribe notas cortas.

## Consulta 1

Que es un archivo README en un proyecto de software?

Notas:

Un archivo README es un documento que explica la informacion principal de un proyecto. Normalmente incluye el nombre, objetivo, tecnologias usadas, instalacion, ejecucion, estructura, funcionalidades, pruebas y observaciones importantes. Sirve para que otra persona pueda entender y ejecutar el proyecto.

## Consulta 2

Que es JSON Server y para que se usa?

Notas:

JSON Server es una herramienta que permite crear una API REST falsa usando un archivo JSON como base de datos. Se usa en proyectos pequenos o academicos para simular un backend y poder guardar, consultar, actualizar o eliminar informacion sin crear un servidor completo.

## Consulta 3

Por que es importante validar formularios?

Notas:

Validar formularios es importante porque evita guardar datos vacios, incompletos o incorrectos. Tambien mejora la experiencia del usuario porque muestra mensajes claros cuando algo debe corregirse. En este proyecto las validaciones ayudan a que los registros guardados sean mas confiables.

## Consulta 4

Que significa documentar pruebas de software?

Notas:

Documentar pruebas de software significa registrar que se probo, que resultado se esperaba, que resultado se obtuvo, que errores aparecieron y que correcciones se aplicaron. Esta informacion demuestra que el proyecto fue revisado de forma organizada.

## Consulta 5

Como se prepara una exposicion tecnica de un proyecto?

Notas:

Una exposicion tecnica se prepara organizando la informacion principal del proyecto: objetivo, problema que resuelve, tecnologias, demostracion del funcionamiento, validaciones, pruebas, errores corregidos, evidencias y aprendizajes. Debe ser clara, breve y apoyarse en una demostracion real.
